# Implementation Plan: Suppress Parent-Device Notifications When Initiating Rewards

**Branch**: `029-suppress-parent-notifications` | **Date**: 2026-04-10 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/029-suppress-parent-notifications/spec.md`

## Summary

When a parent grants a bonus or reward, the notification delivery pipeline currently sends to all FCM tokens registered on the kid's profile — including the token of whichever device the parent is currently using (since the parent and kid often share a device, which registered the kid's FCM token). The fix adds an optional `senderToken` field to the pending notification document. The initiating client includes its own FCM token in this field; the Cloud Function filters it out before sending, so the parent's device is silently excluded while the kid's other devices still receive the notification.

## Implementation Detail

### File 1: `src/firebase/messaging.ts`

**Add** `getSenderToken()`:
```typescript
export async function getSenderToken(): Promise<string | null> {
  if (!VAPID_KEY || VAPID_KEY === 'REPLACE_WITH_YOUR_VAPID_KEY') return null;
  if (Notification.permission !== 'granted') return null;
  if (!('serviceWorker' in navigator)) return null;
  try {
    const swReg = await navigator.serviceWorker.register('/firebase-messaging-sw.js', { scope: '/' });
    return await getToken(messaging, { vapidKey: VAPID_KEY, serviceWorkerRegistration: swReg });
  } catch {
    return null;
  }
}
```

**Update** `writePendingNotification` to accept optional `senderToken`:
```typescript
export async function writePendingNotification(
  familyId: string,
  targetProfileId: string,
  title: string,
  body: string,
  senderToken?: string | null,
): Promise<void> {
  await addDoc(collection(db, 'families', familyId, 'notifications'), {
    targetProfileId,
    title,
    body,
    createdAt: serverTimestamp(),
    ...(senderToken ? { senderToken } : {}),
  });
}
```

### File 2: `src/hooks/useDataProvider.ts`

**Update** the `notify` method in `useCloudDataProvider`:
```typescript
notify: (targetProfileId, title, body) => {
  const fid = familyIdRef.current;
  if (fid && fid !== '__none__') {
    getSenderToken().then(senderToken => {
      writePendingNotification(fid, targetProfileId, title, body, senderToken).catch(() => {});
    }).catch(() => {
      writePendingNotification(fid, targetProfileId, title, body).catch(() => {});
    });
  }
},
```
